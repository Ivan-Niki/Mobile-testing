
Установка приложения имеет решающее значение, и любой недостаток в установке может привести к тому, 
что пользователь отбросит приложение.
Следовательно, тестирование установки приложения необходимо, чтобы убедиться в том, что утсановка проходит ровно и бесперебойно.
Инсталляционное (установчное) тестирование (Application Installation Testing, otherwise known as Application Implementation testing) 
тестирует успешную установку мобильного приложения на различные мобильные телефоны, гаджеты, модели и ОС.
Тестирование установки приложений обычно проводится до того, как приложение впервые взаимодействует со своими клиентами. 
Это последний этап жизненного цикла тестирования программного обеспечения (STLC).
Это гарантирует, что все компоненты вашего приложения полностью согласованны.
Удаление (деинсталляция) приложения не менее важно.
После удаления приложения с портативного устройства важно, чтобы все связанные файлы также удалялись, и это обеспечивается тестированием удаления приложения.

--------- Кейсы проверок установки (первоочередные позитивные кейсы)
- Установка из дистрибутива происходит без уведомлений об ошибках.
- Успешно выполняется установка из магазина приложений.
- Установка обновлений не вызывает ошибок.
- Отмена установки происходит корректно, с удалением всех следов приложения.
- Повторная установка после отмены возможна и выполняется успешно.
- При попытке установки на неподдерживаемое устройство/версию ОС появляется предупреждение о несовместимости, и установка корректно завершается.


Вот шаги для успешного тестирования установки приложения:
-> Убедитесь, что инсталляторы правильно отображают все инструкции.
-> Отмена/остановка процесса установки на полпути, чтобы проверить, возвращается ли система в исходное состояние.
-> Вручную проверьте пространство на диске  и убедитесь, что программа установки приложения правильно рассчитала дисковое пространство.
-> Проверьте реакцию приложения в случае нехватки места на диске. Проверьте, возвращает ли приложение
соответствующее сообщение об ошибке, когда на устройстве мало места на диске.
-> Выполните тестирование удаления приложения, чтобы убедитьтся в удалении всех связанных файлов после того, 
как приложение было удалено.


---------- Типичные тест кейсы при выполнении тестирования установки ----------
-> Проверьте, не установлена ли уже старая версия приложения, и если установлена, сделайте тест кейс для проверки установки приложения по тому же пути.
-> проверка наличия свободного места на диске для установки и включение кейсов для проверки дискового пространства до и сразу после установки.
-> Проверьте на недостаточное место на диске и соответствующее сообщение об ошибке.
-> Убедитесь, что приложение выдаёт соответствующее сообщение об ошибке в ответ на ошибку во время установки.
-> Проверки для принудительной остановки установки.
-> Тест-кейсы для настраиваемого пути установки и пути установки по умолчанию.
-> Тест на низкий объем ОЗУ и соответствующее сообщение об ошибке.
-> Тестирование установки приложений, в условиях высокой безопасности и при включенном брандмауэре.
-> Проверить, как приложение устанавливается, в то время как другое приложение аналогичного типа работает параллельно.
-> Тестирование процесса установки при различных скоростях сети.
-> Проверка установки при нескольких параллельных установках.
-> Проверка времени, необходимого для завершения процесса установки и извлечения.
-> Проверки пути извлечения приложения.
-> Проверки на удаление приложения и наличие тест-кейсов для проверки удаления всех связанных файлов во время удаления.
-> Тест для установки патча и обновления.
-> Проверить реакцию программы установки (инсталлятора) когда имеется более новая версия приложения, а пользователь устанавливает более старую версию.
-> Тестовые проверки для различных прерываний, таких как входящий вызов или сообщение, низкая батарея, подключение зарядного устройства и т.д. во время установки.
-> Тест-кейсы для проверки пути установки как на SD-карте, так и в памяти телефона.


-------- Основные кейсы тестирования обновления ---------

- Обновления устанавливаются корректно.
- При запуске происходит проверка обновлений, и пользователю выдается информация об ограничениях текущей версии.
- Появляются уведомления при нескольких пропущенных обновлениях.
- Появляются уведомления при пропущенных критических обновлениях, препятствующих эксплуатации приложения.
- Приложение работает после обновления ОС.
- Приложение работает после прерванной или отмененной установки обновления.
